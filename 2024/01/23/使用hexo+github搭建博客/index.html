我的个人博客搭建方案

2024年01月23日 1.1k 字 大概 4 分钟

本来想写写如何使用 hexo + Github Pages 搭建博客，然而发现网上的教程已经相当详细，无需再写一篇雷同的文章。因此决定记录搭建此博客中使用的相关工具，希望对有志于搭建个人博客的后来者有所帮助。

博客本体

我采用的是 hexo 来进行博客的搭建。hexo 生成的博客是静态页面，因此需要在本地生成后部署到云端。博客的具体内容需要使用 markdown 进行创作。这里我使用的是 VSCode，可以在写作的过程中预览实际的效果，感觉体验不错。

主题安装

从侧边栏也能看到，我使用的是 hexo-kira 主题来美化我的博客。从 kira-hexo 的官方文档可以看到，安装 kira-hexo 需要使用 npm 或者 yarn 等第三方工具，具体内容可以参考相关的文档。

接下来我将进一步叙述在使用 kira-hexo 主题中遇到的一些问题。

配置文件

参考 kira-hexo 的官方文档，可以发现 kira-hexo 主题和 hexo 本体是有分开的配置文件的。因此，一些设置需要在 hexo 本体的配置文件中进行，另一些则要在 kira-hexo 自己的配置文件中进行。

下面是一些 kira-hexo 配置文件中比较有代表性的项目。

  • iconlib 这一项比较重要，与博客中的图标有关，下文详述。
  • beian 备案号。我留空了这一项。
  • menu 决定了左侧侧边栏显示的项目，我注释掉了“我的朋友”一项。（真是没朋友）
  • social 这里是侧边栏下面的小框，下面也会详细介绍。
  • “讨论区” 这里有 giscus/gitalk 两个可选项，可以根据自己的喜好选择一个使用，也可以两个都不使用。gitalk 可能会遇到一些问题，我使用的是 giscus。
  • copyTip 这里有一个 %url 项，需要在 hexo 本体的配置文件中进行修改，会自动替换。

上传图片

可以使用 picago 来实现利用 Github 来储存图片，这样就可以避免免费图床常见的限制容量、不够稳定等问题了。

关于 iconlib

这一项在 kira-hexo 中没有详细介绍，因此这里的内容主要是我自己摸索出的结果。

在默认情况下，这一项是被注释掉的，此时 kira-hexo 会使用自带的相关图标。如果想要自己添加图标，就需要使用这一项。

然而，如果按注释中的样式，使用阿里云的链接，可能会有不稳定的问题 （大概）。因此，我希望能够打包内容到本地使用。此时，似乎 kira-hexo 就不再使用默认的图标，而只显示了我自行添加的新图标。如何解决这个问题呢？

我的方案是，手动添加所有博客中出现的图标。虽然比较暴力，不过确实生效了。需要注意图标的名称要和配置文件中的相关引用保持一致，否则是无法显示出来的。

侧边栏项目无法访问，显示 404

需要在 source/_posts 文件夹中添加 about.md、archive.md、friends.md 来显示相关的页面。

archive.md

friends.md

about.md 可以直接攥写相关内容。

参考资料

在使用 kira-hexo 主题搭建博客的过程中，官方文档（https://hexo.kira.host/） 是重要的参考。此外，kira-hexo 的前辈 nexmoe 的文档也十分有用，因为这两个主题非常相似，因此配置方式也大同小异。

另外，可以使用 bing、Google 等各种搜索引擎搜索自己需要的相关资料、教程。

身处于 AI 蓬勃发展的时代，ChatGPT 也是能发挥相当作用的。但是要自己具备尝试、修改的能力，方可发挥最大的效力。

碎碎念

虽然博客已经大体成型，但是还有一个功能希望能够添加—— Live 2D 看板娘。由于这个功能相对更为复杂，可能还需要一段时间摸鱼学习才能实现。如果成功的话，我大概也会记录一下实现的过程吧！

就是这样，希望这篇文章对你有所帮助！
